# 更新说明 - 串口手动连接功能

## 💡 新增功能

### 1. 串口自动扫描
- 应用启动时自动扫描系统中的可用串口
- 显示串口号、设备描述、硬件ID
- 支持随时点击"刷新"按钮重新扫描

### 2. 串口手动选择和连接
- 从下拉列表中选择要连接的串口
- 支持修改波特率（9600 ~ 230400）
- 点击"连接"按钮进行手动连接
- 点击"断开"按钮断开连接

### 3. 改进的状态管理
- 连接前采集功能被禁用
- 只有成功连接后才能开始采集
- 实时显示连接状态和错误提示

## 🚀 使用流程

```
1. 启动应用 → 自动扫描串口
   ↓
2. 选择串口和波特率（可选修改）
   ↓
3. 点击"连接"按钮
   ↓
4. 连接成功后点击"开始采集"
   ↓
5. 查看实时曲线图
```

## 📋 控制面板说明

### 串口设置区域
- **选择串口**：下拉列表显示所有可用的COM口
- **波特率**：选择匹配你单片机的波特率
- **刷新按钮**：重新扫描可用的串口设备
- **连接/断开按钮**：控制串口连接状态

### 数据采集区域
- **开始采集**：开始读取并显示数据（需先连接）
- **停止采集**：暂停数据读取（数据保留）
- **清除所有数据**：清空当前缓存的所有数据点

## 🔧 常见操作

### 更换串口
```
1. 点击"断开"按钮
2. 从列表中选择新的串口
3. 点击"连接"
```

### 修改波特率
```
1. 在连接前修改"波特率"下拉框
2. 点击"连接"按钮
```

### 重新扫描串口
```
1. 如果新插入USB设备，点击"刷新"按钮
2. 下拉列表会更新为最新的设备列表
```

## 📊 工作流程改进

### 旧版本（自动连接）
```
启动 → 自动连接 → 自动采集 → 显示数据
```

### 新版本（手动连接）
```
启动 → 扫描串口 → 选择串口 → 手动连接 → 手动采集 → 显示数据
```

## ✨ 优势

1. **灵活性** - 可连接多个串口设备
2. **容错性** - 不会因为找不到特定COM口而启动失败
3. **可靠性** - 波特率不匹配时有明确的错误提示
4. **用户友好** - 图形化界面选择，无需修改代码

## 🎯 应用场景

- **多设备测试** - 快速切换不同的串口设备
- **调试开发** - 重新连接设备无需重启应用
- **跨平台使用** - Windows/Linux/macOS都能自动识别串口
- **教学演示** - 清晰的流程更易理解

## 技术实现

### 后端改进
- 使用 `serial.tools.list_ports` 扫描可用串口
- WebSocket事件 `get_available_ports` 获取设备列表
- WebSocket事件 `connect_port` / `disconnect_port` 控制连接

### 前端改进
- 动态更新串口下拉列表
- 按钮启用/禁用逻辑改进
- 更详细的错误提示信息
