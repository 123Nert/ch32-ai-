# 循迹模块Web可视化应用

一个基于Flask + WebSocket + ECharts的实时数据可视化系统，用于监测和显示循迹模块的ADC数据。

## ✨ 核心特性（v2.0 - 手动连接版本）

**🆕 新增功能：**
- ✅ **串口自动扫描** - 启动时自动检测所有可用串口
- ✅ **手动选择连接** - 灵活选择要连接的串口
- ✅ **波特率可配置** - 支持9600-230400多种波特率
- ✅ **实时连接状态** - 清晰的连接状态显示
- ✅ **错误提示** - 详细的错误和提示信息

**基础功能：**
- 📊 实时ADC值曲线图
- 📈 自动数据统计分析
- 🎮 灵活的控制面板
- 💬 消息日志记录
- 🌐 跨平台浏览器访问

## 完整使用步骤

### 步骤1️⃣：扫描串口

应用启动后，自动扫描系统中的所有可用串口设备，显示在"选择串口"下拉列表中。

如果没有看到你的设备，点击"刷新"按钮重新扫描。

### 步骤2️⃣：选择串口和波特率

```
1. 在"选择串口"下拉列表中选择你的设备
2. 修改"波特率"（如需要，默认115200）
   - 确保波特率与单片机配置相同
3. 检查连接状态应显示"未连接"
```

### 步骤3️⃣：连接到串口

点击"连接"按钮建立与设备的通信连接。

- ✅ 连接成功：状态框变绿，显示"已连接 (COM3)"
- ❌ 连接失败：状态框变红，显示错误信息
  - 检查波特率是否正确
  - 检查设备是否正常工作
  - 尝试重新插拔USB线

### 步骤4️⃣：开始采集数据

连接成功后，点击"开始采集"按钮启动数据读取。

- 曲线图开始显示实时数据
- 统计信息实时更新（当前值、最大值、最小值、平均值）
- 消息日志记录采集进度

### 步骤5️⃣：观察和分析

在曲线图中观察ADC值的变化。

### 步骤6️⃣：停止采集

点击"停止采集"按钮暂停数据读取。

## 数据格式说明

### 串口数据格式

应用期望每行一个数据，格式为整数（0-4095）。

**推荐的单片机代码：**

```c
// 读取ADC值并打印
uint16_t adc_value = LineTracing_ReadADC();
printf("%u\n", adc_value);  // 或其他格式如: printf("ADC: %u\n", adc_value);
```

### WebSocket消息格式

**数据更新消息：**
```json
{
  "time": "14:30:45.123",
  "value": 2048
}
```

## 故障排除

### 问题1：找不到我的设备

**解决方案：**
1. 检查USB设备是否正确连接
2. 点击"刷新"按钮重新扫描
3. 检查驱动程序是否安装

### 问题2：连接失败（波特率错误）

**解决方案：**
1. 确认单片机的波特率配置
2. 修改应用中的"波特率"选项
3. 重新点击"连接"

### 问题3：接收不到数据

**解决方案：**
1. 确认单片机程序正在运行
2. 用串口工具（putty）验证数据格式
3. 检查 `parse_serial_data()` 是否正确解析

2. **压缩数据**：只发送必要的数据
3. **使用更小的MAX_DATA_POINTS**：在 `app.py` 中修改

## API文档

### WebSocket事件

**客户端发送：**
- `connect` - 连接
- `disconnect` - 断开
- `start_capture` - 开始采集
- `stop_capture` - 停止采集
- `clear_data` - 清除数据
- `request_config` - 请求配置

**服务器发送：**
- `connection_status` - 连接状态更新
- `data_update` - 单个数据点更新
- `history_data` - 历史数据
- `capture_status` - 采集状态更新
- `data_cleared` - 数据已清除
- `config_update` - 配置信息
