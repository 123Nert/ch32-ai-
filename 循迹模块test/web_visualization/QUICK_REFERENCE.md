# 快速参考 - 串口手动连接

## 🎯 核心流程

### 第1步：启动应用
```bash
# Windows
双击 run.bat

# Linux/macOS
bash run.sh
```

### 第2步：扫描串口
- 应用自动扫描可用串口
- 或点击"刷新"按钮手动扫描

### 第3步：选择与连接
```
1. 从"选择串口"下拉列表中选择设备
2. 修改"波特率"（可选，默认115200）
3. 点击"连接"按钮
```

### 第4步：开始采集
```
1. 连接成功后，点击"开始采集"
2. 实时曲线图开始显示数据
3. 点击"停止采集"可暂停
```

---

## 📝 使用提示

| 操作 | 说明 |
|------|------|
| 🔄 刷新 | 重新扫描系统中的串口设备 |
| 🔗 连接 | 建立与选定串口的通信连接 |
| 🔌 断开 | 断开当前的串口连接 |
| ▶️ 开始采集 | 启动数据读取（需先连接） |
| ⏸️ 停止采集 | 暂停数据读取（数据保留） |
| 🗑️ 清除数据 | 删除所有采集的数据点 |

## 🔧 常见问题速查

### Q: 找不到我的设备
**A:** 
1. 检查USB设备是否正确连接
2. 点击"刷新"按钮重新扫描
3. 检查驱动程序是否安装

### Q: 连接失败提示波特率错误
**A:**
1. 确认单片机的波特率配置
2. 修改应用中的"波特率"选项
3. 重新点击"连接"

### Q: 数据显示为空白
**A:**
1. 确认单片机程序正在运行
2. 检查数据格式是否符合要求
3. 查看消息日志中是否有错误提示

### Q: 断开连接后无法重新连接
**A:**
1. 点击"断开"按钮
2. 等待1-2秒
3. 点击"连接"重新建立连接

## 🔌 波特率速查表

| 波特率 | 常见设备 |
|--------|---------|
| 9600 | 一些老型号传感器 |
| 19200 | 较少使用 |
| 38400 | 某些工业设备 |
| 57600 | 少数嵌入式系统 |
| **115200** | 👈 最常见（推荐） |
| 230400 | 高速传输 |

> 💡 提示：CH32V30x通常使用115200

## 🎓 单片机串口配置例子

### CH32V30x
```c
// 初始化UART1
void USART_Printf_Init(uint32_t baudrate) {
    // ... 初始化代码 ...
    USART_Init(USART1, &USART_InitStructure);
    USART_Cmd(USART1, ENABLE);
}

// 在主程序中
int main(void) {
    SystemInit();
    USART_Printf_Init(115200);  // 👈 115200波特率
    LineTracing_Init();
    
    while(1) {
        uint16_t value = LineTracing_ReadADC();
        printf("%u\n", value);   // 发送数据
        Delay_Ms(100);
    }
}
```
